# 페르소나 정의 — team-tasks

> **MVP 헌장:** 단일 테이블 · 기능 5개 이하 · 1일 안에 코드로 옮길 수 있는 분량

---

## 페르소나 표

| 역할 | 기술 수준 | 주요 시나리오 |
|------|-----------|--------------|
| **박팀장** — 5인 개발팀 팀장, 스프린트 운영 책임 | 중 — Jira·Notion 경험 있음, 코드는 읽기만 함 | ① 월요일 스탠드업 전 이번 주 일감을 팀원별로 배분한다 ② 마감일이 임박한 일감을 한눈에 파악해 우선순위를 재조정한다 ③ 완료 건수를 집계해 주간 보고 자료에 붙여넣는다 |
| **이개발** — 주니어 백엔드 개발자 2년 차 | 하 — CLI·IDE 익숙, 협업 툴은 시키는 것만 씀 | ① 아침에 접속해 오늘 자기 일감만 필터링해서 확인한다 ② 작업을 시작할 때 상태를 "진행 중"으로 바꾼다 ③ PR 올린 뒤 상태를 "검토 중"으로 바꾸고 설명에 PR 링크를 붙인다 |
| **최대표** — 스타트업 대표, 주 1회 진행 상황 점검 | 하 — 스마트폰 위주, 복잡한 UI 기피 | ① 보드를 훑어 "완료" 컬럼이 늘고 있는지 10초 안에 확인한다 ② 긴급 우선순위 일감이 며칠째 "할 일"에 머무는지 발견하고 팀장에게 슬랙으로 링크를 공유한다 ③ 새 아이디어가 생기면 즉석에서 일감을 만들어 팀장에게 배정한다 |

---

## MVP 범위 도출 (헌장 기준)

페르소나 시나리오를 교집합으로 줄이면 **기능 5개**로 수렴됩니다.

| # | 기능 | 근거 시나리오 |
|---|------|--------------|
| 1 | 일감 생성 (제목·설명·마감일) | 박팀장①, 최대표③ |
| 2 | 담당자 배정 | 박팀장①, 이개발① |
| 3 | 상태 변경 (할 일 → 진행 중 → 검토 중 → 완료) | 이개발②③ |
| 4 | 우선순위 표시 및 마감일 경고 | 박팀장②, 최대표② |
| 5 | 담당자·우선순위 필터 | 박팀장③, 이개발①, 최대표① |

> **범위 밖 (v2 이후):** 댓글, 알림, 권한 관리, 첨부파일, 반복 일감
