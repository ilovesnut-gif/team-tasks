# 유저 스토리 — team-tasks

---

## 1. 핵심 시나리오

**페르소나:** 이개발 — 주니어 백엔드 개발자 2년 차  
**날짜:** 평범한 화요일  

---

### 09:00 — 출근, 오늘 할 일 파악

1. 브라우저에서 team-tasks 앱을 연다.
2. 필터 바에서 자기 아바타(이)를 클릭해 **담당자 = 이개발** 필터를 건다.
3. "할 일" 컬럼에 일감 2개가 보인다.
   - `API 인증 토큰 갱신 로직 구현` — 우선순위 **긴급**, 마감 D-1 경고(빨간색)
   - `모바일 반응형 레이아웃 수정` — 우선순위 낮음
4. 긴급 일감부터 오늘 처리하기로 결정한다.

---

### 09:10 — 작업 시작, 상태 변경

5. `API 인증 토큰 갱신 로직 구현` 카드를 클릭해 상세 다이얼로그를 연다.
6. 설명을 읽고 작업 범위를 확인한다.
7. 상태를 **"할 일" → "진행 중"** 으로 변경하고 저장한다.
8. 카드가 "진행 중" 컬럼으로 이동한 것을 확인한다.

---

### 12:00 — 점심 전 중간 점검

9. 필터가 걸린 채로 보드를 다시 확인한다.
10. 박팀장이 새 일감 `Redis 캐시 TTL 설정 검토`를 추가해 이개발에게 배정한 것을 발견한다.
11. 마감일·우선순위(보통)를 확인하고 오후에 이어서 할 것으로 판단한다.

---

### 13:30 — 오후 작업 재개

12. "진행 중" 컬럼의 `API 인증 토큰 갱신 로직 구현` 카드를 카드 메뉴(···)에서
    **"다음 단계로"** 를 눌러 **"검토 중"** 으로 이동시킨다.
13. 카드를 열어 설명란에 PR 링크를 추가하고 저장한다.
14. `Redis 캐시 TTL 설정 검토` 일감을 **"진행 중"** 으로 변경하고 작업을 시작한다.

---

### 16:00 — 돌발 상황

15. 박팀장이 `API 인증 토큰 갱신 로직 구현`을 검토하다 수정 요청을 남겼다.
16. 이개발은 해당 카드를 카드 메뉴에서 **"이전 단계로"** 눌러 **"진행 중"** 으로 되돌린다.
17. 수정 후 다시 **"검토 중"** 으로 올린다.

---

### 18:00 — 퇴근 전 마무리

18. 박팀장이 PR을 머지하고 `API 인증 토큰 갱신 로직 구현`을 **"완료"** 로 바꾼 것을 확인한다.
19. `Redis 캐시 TTL 설정 검토`는 아직 진행 중 — 설명란에 "내일 오전 마무리 예정" 메모를 추가하고 저장한다.
20. 필터를 초기화해 팀 전체 보드를 훑어보고 퇴근한다.

---

### 하루 요약

| 시간 | 행동 | 사용한 기능 |
|------|------|------------|
| 09:00 | 담당자 필터로 내 일감 확인 | 필터 (담당자) |
| 09:10 | 긴급 일감 → 진행 중 | 상태 변경 |
| 12:00 | 새로 배정된 일감 발견 | 칸반 보드 조회 |
| 13:30 | PR 후 검토 중으로 이동 + 설명 업데이트 | 상태 변경, 일감 수정 |
| 16:00 | 재작업으로 상태 되돌리기 | 상태 변경 (이전 단계) |
| 18:00 | 미완료 일감에 메모 남기고 퇴근 | 일감 수정 |
